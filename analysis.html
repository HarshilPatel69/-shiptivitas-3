Acceptance criterion 3

The SQL queries

Run against shiptivity.db. Both blocks are also in answer.sql.

Part 1 — Daily active users, before & after

-- PART 1a  Daily Active Users time-series (drives the DAU graph)
SELECT DATE(login_timestamp,'unixepoch') AS day,
       COUNT(DISTINCT user_id)           AS active_users
FROM   login_history
GROUP  BY day
ORDER  BY day;

-- PART 1b  Average DAU before vs after the 2018-06-02 release
SELECT period, ROUND(AVG(active_users),2) AS avg_daily_active_users
FROM (
  SELECT DATE(login_timestamp,'unixepoch') AS day,
         CASE WHEN DATE(login_timestamp,'unixepoch') < '2018-06-02'
              THEN 'before' ELSE 'after' END AS period,
         COUNT(DISTINCT user_id)             AS active_users
  FROM login_history
  GROUP BY day
)
GROUP BY period;

Part 2 — Status changes by card

-- PART 2  Number of status changes by card
-- (a real status change = oldStatus changed; exclude creation rows where
--  oldStatus IS NULL)
SELECT h.cardID, c.name, COUNT(*) AS status_changes
FROM   card_change_history h
JOIN   card c ON c.id = h.cardID
WHERE  h.oldStatus IS NOT NULL
  AND  h.oldStatus <> h.newStatus
GROUP  BY h.cardID, c.name
ORDER  BY status_changes DESC;
Acceptance criterion 4

Three bets to move DAU

Each is grounded in a number above and framed as hypothesis → expected impact → feature.

1Activation

Stuck-card nudges

In the data38 of 200 cards (19%) never left the backlog, and the flow leaks from 167 cards started to 103 completed.

Hypothesis
Cards that sit untouched in the backlog are forgotten, not rejected. A timely nudge that resurfaces them will pull the owner back into the app to act.
Expected impact
A measurable drop in never-moved cards and an estimated +10–15% DAU, since each nudge is a reason to open the app that day.
The feature
A weekly “Stale cards” email plus an in-app banner listing cards untouched for 7+ days, each with one-tap move or archive.
2Retention

Completion streaks

In the dataDAU peaked at 12.9 in Jul 2018 then drifted to 11.79 by Feb 2019 — the post-launch novelty is fading.

Hypothesis
Users lose the daily habit once the newness wears off. A lightweight goal tied to the app's core value moment — moving a card to Complete — will rebuild the return loop.
Expected impact
A flatter, slower DAU decline and higher week-over-week return; target +8–12% retained DAU.
The feature
A “cards completed this week” goal and streak counter in the board header, with a gentle reminder when a live streak is about to lapse.
3Acquisition loop

Manager completion alerts

In the dataCompleting a card (in-progress → complete happened 103×) is the value moment managers care about most.

Hypothesis
Emailing a manager the instant their team completes a key card pulls a brand-new, high-value user segment into the app and speeds the whole board along.
Expected impact
A new returning-user segment (managers) converting from email to login; target +15% DAU plus faster card throughput.
The feature
Per-card “notify on complete”: when a card reaches Complete, email the manager a summary with a link that deep-opens the board.
